Структура проекта:
tests/Unit/ - тесты отдельных классов
tests/Integration/ - тесты взаимодействия компонентов
tests/Feature/ - end-to-end тесты
tests/Support/ - вспомогательные классы

Методология "анализ → сопоставление → создание":
Анализируем покрытие в Koseven тестах
Сопоставляем с Modseven API
Создаем современные тесты с нуля

Ключевые отличия в Modseven
Strict typing - все параметры и возвращаемые значения типизированы,
camelCase naming - isArray, isAssoc, setPath вместо snake_case,
Статическое свойство $delimiter для path операций,
Упрощенная логика в некоторых методах,
Единая тестовая функция и одна dataProvider,
Повторно используется проверка на исключение (TypeError) отдельно, без усложнения dataProvider
